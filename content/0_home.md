# ICESat-2 Sea Ice Ecosystem Analysis

## Overview
This Jupyter Book provides analysis and research on ice-based ecosystems using NASA's Ice, Cloud, and Land Elevation Satellite-2 (ICESat-2) data. This project is part of an NSF-funded DeCODER (Data and Computing Opportunities for Discovery and Exploration Research) mini research proposal focused on understanding ecological systems in ice environments.

## Background
Ice-dominated ecosystems, particularly in polar and high-altitude regions, represent unique and fragile environments that are critical to understanding global ecological processes. These ecosystems are experiencing rapid changes due to climate warming, making it essential to develop robust methods for monitoring and analyzing their structure and dynamics.

ICESat-2, launched in 2018, provides unprecedented high-resolution laser altimetry data with meter-scale horizontal resolution and centimeter-scale vertical precision. While primarily designed for ice sheet and sea ice thickness measurements, ICESat-2's capabilities offer new opportunities for characterizing the three-dimensional structure of ice-based ecosystems, including surface topography, ice thickness variability, and habitat characteristics.

## Project Description
The `icey_ecosystems` repository contains code, data, and analysis notebooks for studying ice-based ecosystems using ICESat-2 data. This work aims to:

- Develop methods for extracting relevant data from ICESat-2 altimetry data
- Characterize the spatial and temporal variability of sea ice and ice-based ecosystem structures
- Advance our understanding of ecological processes in ice-dominated environments
- Provide open-source tools and analyses for the broader scientific community

This project contributes to the EarthCube GeoCODES initiative by making ice ecosystem research more discoverable and accessible through standardized metadata and open data practices.

## Using the Book
There are three different ways to use this Jupyter Book:
1. View the rendered Jupyter Notebooks by clicking through the links on the left. Note that there is no interactivity with this option.
2. Run the Notebooks interactively in Binder. Just click the **Binder** tab under the rocket ship icon at the top of each Notebook.
3. Clone (or fork then clone) the GitHub repository and run the Jupyter Notebooks locally. Just click the **repository** tab under the GitHub icon at the top of this page.
